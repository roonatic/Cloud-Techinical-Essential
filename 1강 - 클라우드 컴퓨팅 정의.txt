<1강 - 클라우드 컴퓨팅 정의>

[개요]
미국 국립 표준 기술원은 5대 주요 특징,  3개 서비스 모델과, 4개 배치 모델에 기초하여 개념을 정의

정의)
최소한의 관리 노력이나 최소화된 서비스 제공업체와의 상호 작용으로도 신속하게
프로비저닝 및 릴리스 등의 설정이 가능한 컴퓨터 리소스 공유 풀(소프트웨어, 스토리지, 서버,
네트워크)에 어디서든 편리한 주문형 네트워크를 통해 액세스를 가능하게 하는 모델

5가지 특징)
* 빠른 탄력성(Rapid Elasticity)
 - 사용 및 삭제 요청시 빠른 조치 가능
 - 수 초~수 분
* 주문형 셀프 서비스(On Demand Self-Service)
 - 사용자가 직접 원하는만큼 신청하여 사용
 - 웹 기반 GUI, CLI 등 다양한 형태의 사용 환경 제공
* 자원의 공통관리(Resource Pooling)
 - 각 개별 자원들을 통합하여 중앙 관리되는 형태로 구성
 - 가상화 환경 기반
* 광대역 네트워크 접근(Broad Network Access)
 - 인터넷 같은 네트워크망을 통하여 자원을 사용
 - 어디서든 사용이 용이
* 측정 가능한 서비스(Measured Service)
 - 각 사용자별 자원 사용량 측정 가능
  - 과금 측정, 모니터링 제공

3가지 서비스 모델)
* IaaS(Infrastructure as a Service)
 - 서버, 네트워킹, 데이터 저장 등의 인프라 제공
 - 손쉽게 이전하거나 기존 온프레미스 환경과 함께 사용 가능
* PaaS(Platform as a Service)
 - 개발 환경 등의 플랫폼을 제공
 - 조직에서 기본 인프라를 관리하는 필요성 제거
* SaaS(Software as a Service)
 - 소프트웨어 제공
 - 서비스 제공자가 운영 및 관리

 4가지 배포 모델(Deployment Models)
 * Private Cloud
  - 특정 조직이 사용할 수 있도록 구성
  - 사내 또는 외부에 배포 가능
  - 예) 기업 내부 직원
 * Community Cloud
  - 특정 그룹이 사용할 수 있도록 구성
  - 사내 또는 외부에 배포 가능
  - 예) 미국 Gov Cloud
 * Public Cloud
  - 불특정 다수가 사용할 수 있도록 구성
  - 일반 대중 대상
  - 클라우드 공급자 환경에 배포
  - 예) AWS, Azure 등
 * Hybrid Cloud
  - 2개 이상의 배포 모델 병행 사용
  - 서로 데이터, 애플리케이션 이식 가능
  - 예) Private + Hybrid
